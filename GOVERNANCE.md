# Governance

Resource skills should stay portable. They may define generic resource-store
contracts, but they must not encode one workspace's project, client, knowledge,
or archival taxonomy.

## Design Rules

- Put resource-root structure in `resource-scaffold`.
- Make other skills depend on `resource-scaffold` instead of creating their own
  folder trees.
- Keep ingestion separate from indexing. Ingestion places or stages files;
  indexing detects file changes, refreshes metadata, and updates embeddings.
- Treat clean and messy drops as modes of one ingestion skill.
- Delegate document parsing to document skills instead of duplicating OCR,
  DOCX, PDF, or table extraction logic here.
- Keep media support open-ended: videos, audio, and transcripts should be
  derivatives linked to source records, not a separate corpus model.
- Prefer local, reproducible, file-backed stores before external services.

## Backend Direction

The default backend should be Python-managed and local:

- file state tracked by stable path, size, mtime, content hash, and status
- resource records stored in a local non-relational/vector store
- chunk embeddings generated by a configurable Hugging Face model
- client/workspace metadata defaults and path rules loaded from configuration,
  not hard-coded into portable skills
- sync jobs idempotent across repeated runs

LanceDB is the preferred first backend. Alternate profiles can be added later
only when they preserve the same scaffold contract.
